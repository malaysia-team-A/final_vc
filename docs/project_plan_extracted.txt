UCSI INTERNSHIP AI CHATBOT 기획안 추출 정리본
기준일: 2026-02-09

1) 프로젝트 목표
- 2주(총 40시간) 내 Python 기반 AI 챗봇 프로토타입 구축
- 입학/학과/교수/기숙사/학사정보 질의를 통합 지원
- UCSI 데이터베이스와 연동해 사실 기반 응답 제공

2) 핵심 방향
- 짧은 개발 주기 내 동작 가능한 MVP 완성
- 팀 단위 병렬 개발(기능 분할)
- 사용자 피드백을 통한 점진적 품질 개선

3) 권장 아키텍처(기획 원문 반영 + 현재 구현 동기화)
- API 계층: FastAPI
- 인증: JWT 기반
- DB 계층: MongoDB Atlas
- 검색: 벡터 검색(FAISS) + 규칙/키워드 결합
- AI 계층: LLM 기반 응답 생성
- 관리 계층: 문서 업로드/지식 갱신

4) 기능 범위
- 일반 질의: 인증 없이 학교 일반 정보 제공
- 개인 질의: 로그인 후 본인 정보 조회
- 민감 질의: 비밀번호 재검증 후 성적/GPA 조회
- 관리자 기능: 지식 파일 업로드, 통계/미응답 관리

5) 자기개선(Self-learning) 정의
- 완전 자율 학습이 아니라, 운영자가 통제 가능한 개선 루프
- 미응답/오답 로깅 -> 데이터 보강 -> 재색인 -> 품질 향상
- 피드백(긍정/부정)을 반영한 응답 경로 개선

6) 보안/컴플라이언스
- 성적 조회 이중 인증(로그인 + 비밀번호 검증)
- DB 읽기 중심 접근
- 로그 개인정보 최소화/익명화 권장

7) 최종 산출물
- 실행 가능한 챗봇 서버
- DB 연동 데모
- 관리자 지식 관리 기능
- 검증 결과 리포트 및 기술 문서

8) 현재 코드 대비 메모
- 현재 구현은 FastAPI 비동기 구조로 전환 완료
- 포트는 8000을 기본 사용
- QA 스크립트와 리포트 경로가 프로젝트에 포함되어 있음

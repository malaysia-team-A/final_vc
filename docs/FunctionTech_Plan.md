# 기능 확장 계획 (Function + Tech)

- 기준일: 2026-02-09
- 목표: 현재 FastAPI 비동기 구조를 유지하면서 운영 안정성과 사용자 품질을 동시에 개선

## 1. 단기 개선 (2주)
1. 관리자 보안 강화
- `/api/admin/*` 인증/권한 체계 추가
- 업로드/삭제 작업 감사 로그 추가

2. 응답 품질 정리
- 인코딩 깨짐 문자열 정리
- 한국어/영어 응답 톤과 문구 일관화

3. 모니터링 보강
- 오류 유형별 카운트(인증 실패, NO_DATA, rate-limit)
- 관리자 페이지에 기본 지표 추가

## 2. 중기 개선 (1개월)
1. 검색 품질 향상
- query rewrite 트리거 정교화
- label 기반 검색 가중치 튜닝
- 컬렉션 alias 정리로 검색 누락 감소

2. 세션/상태 관리 고도화
- 현재 메모리 세션(`high_security_sessions`)의 만료/동시성 정책 강화
- 필요 시 Redis 도입 검토

3. QA 자동화
- strict/stress 결과 임계치 기반 실패 처리
- 회귀 테스트셋 자동 갱신 워크플로우 구축

## 3. 장기 개선 (분기)
1. 관리자 운영 기능 확장
- unanswered 대응 워크플로우(해결/보류/폐기)
- 피드백 기반 지식 반영 승인 절차

2. 제품 신뢰성
- 답변 근거 가시화(출처 카드)
- 모델 fallback 전략 다중화

3. 개발 생산성
- 레거시 동기 엔진 단계적 제거
- 모듈 책임 경계 명확화(API/엔진/정책)

## 4. 성공 지표
- 보안: 관리자 API 무인증 접근 0건
- 품질: Strict QA 95% 이상 유지
- 안정성: Stress P95 지연 개선 추세
- 운영: unanswered 처리 리드타임 단축
